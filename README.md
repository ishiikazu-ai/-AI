# 業績達成サポートAI（Phase 2）

## プロダクト概要

社員の業績達成をAIがコーチング・メンタリングで支援するプロダクト。
3つのPhaseで構成される人事評価支援AIの中核。

### ロードマップ

| Phase | 内容 | ステータス |
|-------|------|-----------|
| Phase 1 | 目標設定支援AI（AppSheet + GAS + Gemini） | 稼働中・100名超利用 |
| Phase 2 | 業績達成サポートAI | **開発中** |
| Phase 3 | 評価支援AI | 企画段階 |

---

## Phase 2 設計方針

### コンセプト
- **数字管理ツールではなく、対話ツール**
- 社員にKPIの数値入力を求めない（タレントパレットとの二重入力を回避）
- AIが目標データを把握した上で、自然な対話でコーチング/メンタリングを実施
- 対話ログをPhase 3（評価支援AI）の入力データとして蓄積

### AIの振る舞い
- **最初の声がけ**：AIが社員のKPI・ミッションの具体的な数字を提示し「全部知っていますよ」と示す → 社員の意識づけ
- **コーチングモード**（通常時）：進捗の手応えを聞き、うまくいっていることを引き出す
- **メンタリングモード**（壁にぶつかっている時）：AIが察知して切り替え、寄り添いながらヒントを提供
- **コンディション確認**：対話の終盤でさりげなく体調・気持ちを確認 → アラート機能

### アクセス権の設計
- **社員**：自分のデータのみ閲覧可能（AppSheet Security Filter）
- **HR**：全社員のデータを閲覧可能
- **上司**：直接アクセス不可。HRが必要に応じて別途報告

## 技術スタック

| 要素 | 技術 | 役割 |
|------|------|------|
| UI | AppSheet | 対話画面・認証・データ閲覧 |
| AI | Gemini（GAS連携） | コーチング/メンタリング対話 |
| ロジック | GAS (Google Apps Script) | Gemini呼び出し・メール配信・ログ書き込み |
| DB | Google スプレッドシート | 目標データ・対話ログ・月次サマリー |
| 通知 | Gmail（GAS経由） | 月1回の進捗確認案内メール + AppSheet URL |
| 目標データ元 | タレントパレット | CSVエクスポート → スプレッドシートに取込 |

### 技術的制約
- 社内ポリシーによりAPIは使用不可 → GASの設定でGeminiと接続
- Slack連携はエンジニアリング本部への依頼が必要なため見送り → メール配信方式を採用
- 開発者はHR本部所属（非エンジニア）→ バイブコーディングで開発

## データ構造（スプレッドシート 4シート構成）

### シート①：社員目標マスタ
タレントパレットからのエクスポートデータを格納。AIがプロンプト参照用に使用。社員は編集しない。

| 主な列 | 説明 |
|--------|------|
| 社員番号・氏名・等級・部署 | 基本情報 |
| メールアドレス | Security Filter・メール配信に使用 |
| 今期ミッション | 大きな使命 |
| KPI内容①〜⑧ / 当月目標①〜⑧ | 業績KPI（最大8つ） |
| スキル成長目標①② / 目標詳細 / 実現方法 | スキル目標 |
| 職能要件 / 求められるスキル | 次等級の要件 |

### シート②：対話ログ
AppSheetの対話画面でのやり取りを1行ずつ蓄積。

| 列 | 説明 |
|----|------|
| ログID | 一意のID |
| セッションID | 月単位のセッション識別子 |
| 社員番号 | 社員の識別 |
| タイムスタンプ | 発話日時 |
| 発話者 | AI / 社員 |
| メッセージ本文 | やり取りの内容 |
| AIモード | コーチング / メンタリング |

**運用ルール**：800人×月10往復で月間約16,000行。月末にGASで前月分を別スプレッドシートにアーカイブ。

### シート③：月次サマリー
対話終了時にGeminiが自動生成する要約。HRが全社員分を閲覧。

| 列 | 説明 |
|----|------|
| 総合ステータス | 順調 / 要注意 / 要フォロー（Gemini自動判定） |
| コンディション | グリーン / イエロー / レッド（Gemini自動判定） |
| ①KPI・ミッション進捗感 | ざっくりした進捗の手応え |
| ②課題・ボトルネック | 壁になっていること |
| ③うまくいっていること | ナレッジとして蓄積 |
| ④スキル成長の状況 | スキル目標の進捗 |
| ⑤KPI外の貢献・気づき | 目標外の貢献 |
| ⑥コンディション詳細 | 体調・気持ちの状況 |
| HRメモ | HR担当が手動記入（ケア面談記録など） |

**アラート機能**：コンディションが「レッド」の場合、GASが自動でHR担当にアラートメールを送信。

### シート④：配信管理
メール配信の管理。

| 列 | 説明 |
|----|------|
| 対話ステータス | 未配信 / 配信済・未回答 / 対話中 / 完了 |
| 次回配信予定日 | GASトリガーの判定に使用 |

## 全体フロー

```
月初：GAS時間トリガー
  ↓
Gmail経由で全社員にメール配信
「今月の業績進捗確認をお願いします」+ AppSheet URL
  ↓
社員がAppSheetを開く
  ↓
社員番号認証 → 本人データだけ表示
  ↓
対話画面でAIと会話
  - AIが最初にKPI・ミッションの数字を提示
  - コーチングモードで進捗確認
  - 必要に応じてメンタリングモードに切替
  - 終盤でコンディション確認
  ↓
対話終了 → Geminiが月次サマリーを自動生成
  ↓
コンディション「レッド」→ HR自動アラート
  ↓
HRが月次サマリーを確認 → 必要に応じて上司に報告・ケア面談実施
```
