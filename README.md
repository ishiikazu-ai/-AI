# 業績達成サポートAI（Phase 2）

## プロダクト概要

社員の業績達成をAIがコーチング・メンタリングで支援するプロダクト。
3つのPhaseで構成される人事評価支援AIの中核。

### ロードマップ

| Phase | 内容 | ステータス |
|-------|------|-----------|
| Phase 1 | 目標設定支援AI（AppSheet + GAS + Gemini） | 稼働中・100名超利用 |
| Phase 2 | 業績達成サポートAI | **プロトタイプ完成** |
| Phase 3 | 評価支援AI | 企画段階 |

---

## Phase 2 設計方針

### コンセプト
- **数字管理ツールではなく、対話ツール**
- 社員にKPIの数値入力を求めない（タレントパレットとの二重入力を回避）
- AIが目標データを把握した上で、自然な対話でコーチング/メンタリングを実施
- 対話ログをPhase 3（評価支援AI）の入力データとして蓄積

### AIの振る舞い
- **最初の声がけ**：AIが社員のKPI・ミッションの具体的な数字を提示し「全部知っていますよ」と示す → 社員の意識づけ
- **コーチングモード**（通常時）：進捗の手応えを聞き、うまくいっていることを引き出す
- **メンタリングモード**（壁にぶつかっている時）：AIが察知して切り替え、寄り添いながらヒントを提供
- **コンディション確認**：対話の終盤でさりげなく体調・気持ちを確認 → アラート機能

### 画面構成
- **社員向け**：HTMLチャット画面（GAS Webアプリ）— メールのURLからアクセスしてAIと対話
- **HR向け**：AppSheet — 月次サマリー閲覧、コンディション確認、HRメモ記入、配信管理
- **上司向け**：未実装（今後必要に応じて開発）

### アクセス権の設計
- **社員**：チャット画面で自分の対話のみ。AppSheetでは自分のデータのみ閲覧可能
- **HR**：AppSheetで全社員のデータを閲覧可能
- **上司**：直接アクセス不可。HRが必要に応じて別途報告

## 技術スタック

| 要素 | 技術 | 役割 |
|------|------|------|
| 対話UI | GAS Webアプリ（HTML/CSS/JS） | チャット形式の対話画面 |
| 管理UI | AppSheet | HR向け管理画面（サマリー閲覧・HRメモ） |
| AI | Gemini 2.5 Flash（Vertex AI経由） | コーチング/メンタリング対話 |
| ロジック | GAS (Google Apps Script) | Gemini呼び出し・メール配信・ログ書き込み |
| DB | Google スプレッドシート | 目標データ・対話ログ・月次サマリー |
| 通知 | Gmail（GAS経由） | 月1回の進捗確認案内メール + チャットURL |
| 目標データ元 | タレントパレット | 一括ダウンロード → スプレッドシートに取込 |

### 技術的制約
- 社内ポリシーによりAPIは使用不可 → GASの設定でGeminiと接続（ScriptApp.getOAuthToken()）
- Slack連携はエンジニアリング本部への依頼が必要なため見送り → メール配信方式を採用
- 開発者はHR本部所属（非エンジニア）→ バイブコーディングで開発

## 全体フロー

```
月初：GAS時間トリガー
  ↓
Gmail経由で全社員にメール配信
「今月の業績進捗確認をお願いします」+ GAS WebアプリURL
  ↓
社員がURLをクリック → チャット画面が開く
  ↓
Googleアカウント認証 → メールアドレスで社員を自動識別
  ↓
「進捗確認を始める」ボタンを押す
  ↓
AIが最初にKPI・ミッションの具体的内容を提示してコーチング開始
  - コーチングモードで進捗確認
  - 必要に応じてメンタリングモードに切替
  - 終盤でコンディション確認
  ↓
「対話を終了」ボタン → Geminiが月次サマリーを自動生成
  ↓
コンディション「レッド」→ HR自動アラートメール
  ↓
HRがAppSheetで月次サマリーを確認 → 必要に応じて上司に報告・ケア面談実施
```
